Get Started

Installing and configuring the plugin is simple. Follow the steps below to integrate `live-server.nvim` into your workflow.

Using Lazy.nvim

Place the `live-server.nvim` directory in your config path and add the following spec to your `lazy.nvim` setup.

{
  'G00380316/live-server.nvim',
  lazy = false,
  config = function()
    require("live-server.nvim").setup({})
  end,
}

Default Configuration

Pass a configuration table to the `setup()` function to override the defaults. Most users won't need to change anything.

require("live-server.nvim").setup({
  browser_sync_port = 3000,
  live_server_port = 8080,
  files_to_watch = '"*.html, *.css, *.js"',
  auto_open_browser = true,
})

Recommended Keymaps

For the best experience, add these keymaps to your `lazy.nvim` spec to quickly access the most common features.

keys = {
  {
    "sl",
    function() require("live-server.nvim.core").toggle_live_server() end,
    desc = "Toggle Live Server"
  },
  {
    "sb",
    function() require("live-server.nvim.core").toggle_browser_sync() end,
    desc = "Toggle BrowserSync"
  },
  {
    "sL",
    function() require("live-server.nvim.ui").list_servers() end,
    desc = "List Live Servers"
  },
}

Functional Overview

The plugin's commands are designed for specific tasks. This chart provides a high-level breakdown of the available functionality.

Toggle Commands

Start and stop servers. The core functionality of the plugin.

  • :LiveServerToggle [port]
  • :BrowserSyncToggle [port]

UI Commands

Interact with the plugin through floating windows and prompts.

  • :LiveServerList
  • :LiveServerPrompt
  • :BrowserSyncPrompt

Utility Commands

Convenience commands for common actions like opening a browser tab.

  • :LiveServerOpen
  • :BrowserSyncOpen
